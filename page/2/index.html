前端开发学习笔记 14 背景与精灵图

发表于 2021-03-20 更新于 2023-03-25 分类于 笔记
本文字数： 1.2k 阅读时长 ≈ 4 分钟

背景

背景颜色

transparent 不要忘记

背景图

任何元素，都可以有背景图
背景图的设置要通过background-image:url();
其中 url 就是图片路径

背景重复

background-repeat

  • repeat 默认，背景图将会在 x 轴 y 轴上重复
  • no-repeat 不重复，背景图只显示一次
  • repeat-y 背景图只在 y 轴重复
  • repeat-x 背景图只在 x 轴重复

背景位置

控制背景图从哪里开始显示
background-position:x y;
常见的是像素值，但是实际上我们更常用的
top right left bottom center
可以只写一个值，此时另外一个值默认为center

背景固定

background-attachment

  • scroll 默认值，背景图会跟随页面滚动
  • fixed 背景图固定，常常用来做视差滚动

背景尺寸

background-size:<length>|%|||

  • length 第一个值为水平宽度，第二个值为垂直高度，当只写一个值的时候，另一个为 auto
  • % 也是两个值，是相对于父元素的百分比
  • cover 让图片完全覆盖背景，这个属性一般会让背景图显示不全
  • contain 会把图片缩放到合适大小，让图片完全显示，一般会让部分区域空白

复合写法

background: 颜色 路径 重复？ 位置/尺寸

  • 背景尺寸必须紧跟在背景位置的后面，两者之间用一条/分隔
  • 值的规则仍然适用，横向的值在前，纵向的值在后

一些注意事项

  1. 背景图和背景颜色，从哪里开始显示?(边框？内边距？)
    背景颜色从边框区域开始渲染,而背景图片从内边距开始渲染
    某些情况下，需要将背景图渲染的位置发生改变，我们就可以用到
    background-origin:border-box|padding-box|content-box
  • border-box:图像从边框区域开始
  • padding-box:图像从内边距开始(默认值)
  • content-box:图像从内容区域开始

2.什么时候用 img？什么时候用 bgi?

  • img
    img 标签虽然不是块级元素，但是他可以设置宽高，占位
    img 设置宽度以后，高度会自动匹配，如果设置高度那么宽度也一样，如果同时设置宽高，图片可能会变形。

  • bgi
    他只是一个 css 属性，并不占位，可以结合其他属性来设置背景图的大小，位置，重复与否。她所在的元素必须设置宽高

他们的不同

  1. img 是标签占位；bgi 不是标签不占位
  2. img 是 docment 对象中可以操作的，但是 bgi 不行
  3. 在网页的加载过程中，img 是结构的一部分，会在加载结构的时候加载，而 bgi 是样式，它会在结构加载完成以后再加载
  4. img 标签点击会选中/放大图片，bgi 不会

使用 img

  • 作为结构
  • 展示后台传过来的数据
  • 对图片进行缩放
  • seo 搜索引擎优化
  • 对语义化有明确要求
  • 图片是作为页面的组成部份而不是修饰部份
  • 显示图片只给定宽/定高 而且不能截断的时候

使用 bgi

  • 不是结构的一部分
  • 图像代替文本使用
  • 禁止用户缩放时
  • 允许图片 拉伸 重复平铺 截取

雪碧图

雪碧图是一种网页处理图片的方式，它允许将一个页面涉及到的所有零星的图片都整合到一张大图中。当访问该页面的时候，载入的图片就不会像往常那样慢慢显示出来。

作用

简单来说，使用雪碧图的主要目的就是为了有效的减少 Http 请求数量和加速内容的显示

雪碧图的使用规则

虽然 css sprite 能够优化网络资源的加载情况，但是并不是每个图片适合用雪碧图，必须满足以下条件

  • 静态图片，不会因为用户的差异性而发生变化
  • 小图片，图片的体积比较小，对清晰度要求不高
