3. 作用域插槽

3.1 渲染作用域

父级模版里面的所有内容都是在父级作用域中编译的

子模版里面的所有内容都是在子级作用域中编译的

案例：在NavBar中可以访问自己作用域的title，但是在App中的h2无法访问，因为他是跨作用域的

但是，我们往往又希望插槽可以访问到子组件中的内容，这个时候就得用到作用域插槽

3.2 作用域插槽

作用域插槽就是为了解决3.1的问题，让插槽可以访问子组件中的内容

  1. 在App.vue中定义好数据

  2. 传递给子组件

  3. 在子组件中使用数据

  4. 定义插槽的props

  5. 通过v-slot:default的方式获取到插槽的props

  6. 使用插槽的props拿到子组件中的内容

插槽能够访问到子组件中的内容，本质是把slot当作一个组件传递props

3.3 独占默认插槽的缩写

如果我们的插槽是默认插槽default，那么v-slot:default=”xxxxx”可以简写为v-slot=”xxxxx”

4. 非父子组件通信

在实际开发中父子组件之间的通信往往选择props和emits,除了父子之外我们还有一些其他的关系组件需要通信，比如爷孙，比如兄弟,甚至还有关系不太好描述的组件.

两种：全局事件总线bus,provide/inject

4.1 全局事件总线

在Vue2中是自带的，但是Vue3没有了移除了$on,$off,$once…方法，如果我们希望继续使用事件总线，要通过第三方的库。

mitt

  1. 安装 npm install mitt -S

  2. 配置 在src/utils/bus.js中写入如下内容

  1. 开始使用，就可以在任意关系的.vue文件之间传递数据

App.vue

StudentCom.vue

虽然我们是爷爷和孙子通信但是这种bus的通信方式，任意关系的组件均可

发送数据 bus.emit(‘事件名称’,数据)

接收数据bus.on(‘事件名称’,(数据)=>{…})

接收所有的数据 bus.on(‘*’,(type,data)=>{type具体的某一个事件，data该事件的数据})

停止监听事件 bus.off(‘事件名称’)
