组件化开发

1.组件化开发的思想**

1.1 认识组件化开发

组件化开发也是类似的把大问题转成很多个方便解决的小问题，比如如果将所有的逻辑全部放在一起，处理起来变得非常复杂，而且不利于后续的维护和扩展。但是如果我们把页面拆成一个个小的功能块，每个功能块完成自己独立的部分，整个页面的管理就变得非常容易。

学习组件化开发最重要的是他的思想，每个框架，平台具体的实现不同，但是思想都是一样的。

组件最大的好处是方便复用。

1.2 Vue的组件化开发

组件化是Vue的核心思想，也是课程的重点内容

​ 前面我们的createApp函数传入了一个配置对象App，这个对象本质上就是一个组件，而且也是整个应用程序的根(Root)组件

组件化开发，简单来说就是两步

1.3 Vue的SFC

真实开发中，我们会通过一个后缀名为.vue的单文件组件(Single File Component)来解决，并且通过统一的开发结构；vue-cli/vite/webpack/rollup。

使用sfc的.vue文件来开发

  2. 自己用webpack或者vite来处理

2. VueCLI

2.1 什么是VueCLI脚手架？

真实开发我们不可能每一个项目都从头来配置webpack/vite，这样开发效率会非常的低，所以真实开发我们通常会使用脚手架(帮助我们搭建好一些基础结构)来创建项目，Vue项目所使用的就是Vue的脚手架即Vue CLI

​ CLI：Command Line Interface 命令行界面

​ vue-cli已经内置了webpack的配置，不需要从0开始

2.2 Vue CLI的安装和使用

安装：npm install @vue/cli -g

升级: npm update @vue/cli -g

使用：vue create 项目名

更多： https://cli.vuejs.org/zh/

Windows用户，vue-cli的配置文件在一个叫做**.vuerc**的文件上,C:\Users{当前用户名}

browserslist文件：做浏览器适配

jsconfig.json: 给vscode用的

额外补充：tree

windows可以直接使用：

mac需要安装tree，homebrew install tree

如果报错得先安装homebrew

命令：tree > tree.md 这样你的整个项目目录会被输出到当前的tree.md文件中

忽略node_modules文件夹的目录输出：tree -I ‘node_modules’ > tree.md

其他命令自行百度命令行tree的使用

3.2 组件自己的逻辑

组件也可以有自己的代码逻辑，比如自己的data，methods，computed…

再创建一个组件product-item

scoped样式隔离：如果style标签不加scoped隔离，会导致样式穿透，那个组件后引入就应用对应的样式，哪怕之前有同样的选择器，自己的样式也不生效。加一个scoped就能解决，为什么？

原理： 加了scoped以后每个组件的内的模版中的所有标签都加上了一个 **data-v-[hash:8]**，最终你的style样式应用上去的时候会自动加上一个对应的属性选择器div[data-v-hash]{}

为什么data必须是一个函数：让每个组件都有自己的独立的数据。

3.4 局部组件

全局组件往往会在应用程序一开始就注册，这样有些组件即使没有使用到，也会注册

  1. 打包后的js体积会增加，不利于webpack的tree shaking

  2. 使得依赖关系的维护很困难，而且不太容易定位子组件的实现，以及过多的全局变量

所以开发中除非明确是全局都需要的一般都会采用局部组件的形式

使用局部组件操作如下

  1. 编写并且注册，通过选项API，components选项来注册

  2. components中的选项对应的是一个对象，对象中key:组件的名称，value：组件的实现
