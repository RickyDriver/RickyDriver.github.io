事件

1.了解事件

什么是事件？

  • 我们使用 代码的方式 和 一个内容 约定好一个行为，当你打开浏览器，触发了该行为，会有对应的代码执行

事件的三要素

  1. 事件源 和谁约定事件 (和由谁触发不一样)
  2. 事件类型 约定了一个什么样的事件
  3. 事件处理函数 当事件发生时，需要执行的函数

这三个内容，目的是为了绑定事件，注册事件

事件的绑定

  1. DOM0 级事件
    事件源.on 事件类型=事件处理函数

给一个事件源的同一个事件类型，只能绑定一个事件处理函数

  1. DOM2 级事件(事件侦听器/事件监听器)
    事件源.addEventListener(‘事件类型’,事件处理函数)
    可以给同一个事件源的同一个事件类型绑定多个处理函数

2. 事件解绑

  1. DOM 0 级的解绑
    事件源.on 事件类型 = null
  1. DOM 2 级事件解绑
    事件源.removeEventLister(‘事件类型’,事件处理函数)

3. 事件类型

  1. 鼠标
    click 鼠标左键单击
    dbclick 鼠标左键双击
    contextmenu 鼠标右键单击
    mousedown 鼠标按下
    mouseup 鼠标抬起
    mousemove 鼠标移动
    mouseover 鼠标移入
    mouseout 鼠标移出
    mouseenter 鼠标移入
    mouseleave 鼠标移出
    over/out 一套不光移入事件源触发，移入子元素也触发
  2. 键盘
    keydown 键盘按下 任何一个按键都能触发
    keyup 键盘抬起 任何一个按键都能触发
    keypress 一直按住 必须要按下可以真实键入内容的按键或者回车键才能触发 ctrl shift alt win
  3. 浏览器的
    DOMContentLoaded 浏览器的已经完全加载了 html，并且构建完了 DOM 树
    load 浏览器不仅完成了 HTML，还加载完了所有外部资源，img css
  4. 表单的
    focus 聚焦
    blur 失去焦点
    change 改变 要求聚焦和失焦 时内容不一致才会触发
    input 只要表单元素输入内容或者删除内容 就会触发
    reset 事件都是给 form 标签的由 form 标签内的 reset 按钮触发
    submit 事件都是给 form 标签的由 form 标签内的 sumbit 按钮触发
  5. 触摸
    只能在移动端使用
    touchstart 触摸开始：手接触到屏幕的瞬间
    touchmove 触摸移动
    touchend 触摸结束
  6. 其他
    selectstart
    transitionend
