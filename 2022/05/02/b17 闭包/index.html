闭包

闭包是函数的高级使用方式

函数

函数的特点

  1. 保护私有变量
    因为每一个函数都会生成一个独立的作用域
    在函数内部定义的变量 叫 私有变量
    该变量只能在函数作用域及下级作用域内使用，外部无法使用
  1. 函数定义时不解析变量
    函数定义的时候，函数体内的代码完全不执行
    任何变量不做解析
    直到执行的时候才会解析变量

函数的两个阶段

  1. 定义
  2. 调用

函数的两个阶段分别做了什么事(记住)

阶段一:定义

  1. 在 堆内存 中开辟一段存储的空间
  2. 把函数体内的代码，一模一样的复制一份，以字符串的形式放进这个空间，此时不解析变量
  3. 把 堆内存 中的空间地址赋值给变量

阶段二:调用

  1. 按照变量名(函数名)内的地址寻找到 堆内存中对应的空间
  2. 在 调用栈 内开辟一段新的函数执行空间
  3. 在 新的执行空间 进行形参赋值
  4. 在 新的执行空间 进行预解析
  5. 在 新的执行空间内 把拿到的函数体代码执行一遍
  6. 把 开辟的调用栈 内的 执行空间 销毁(等到你函数内的所有代码执行完毕)

函数执行空间

  • 每个函数都会有一个存储的空间
  • 但是每一次调用函数都会生成一个完全不一样的执行空间
  • 并且执行空间 会在函数 执行完毕后就销毁，这和存储空间不一样

我们有一些办法让这个执行空间不被销毁
闭包，就是要利用这个 不销毁的执行空间

函数执行空间不销毁

  • 函数的 执行空间 会在函数执行完毕后 销毁
  • 但是，一旦函数内部返回了一个 引用数据类型，并且 在函数外部有变量接收 的情况下
  • 这个执行空间 就不会销毁了


函数执行的时候，会生成一个函数执行空间，我们叫他 0x100-1
代码在 0x100-1 空间中执行
在 0x100-1 这个空间中声明了一个对象空间(0x200)
在 0x100-1 中把对象 obj 的地址返回
函数外部的 o 接收的是一个对象的地址
但是这个地址是在 0x100-1 的函数执行空间中的对象的地址
因为 o 变量一直在和这个对象的地址关联。所以 0x100-1 这个执行空间一直不会销毁

等到什么时候，执行了一句o=null
此时，o 变量里面存的地址不再和 0x100-1 函数执行空间关联了
这个时候，函数执行空间就销毁

闭包

闭包就是利用 这个函数执行空间不销毁的逻辑
有几个条件才能组成闭包

不销毁空间

  • 闭包的第一个条件就是利用不销毁的空间
  • 他返回的只不过不是一个对象
  • 而是返回一个函数数据类型

f 变量接收的就是一个 fn 执行空间中匿名函数的地址

内部函数引用外部函数中的变量

  • 涉及到两个函数
  • fn()会生成一个 xxff00 函数执行空间
  • 在 xxff00 这个执行空间内部，定义了一个 a 函数的存储空间 xxff11
  • 全局变量 f 接收到的 xxff00 里面的 xxff11
  • 所以 xxff00 不会销毁
  • 因为他不会销毁，所以里面定义的 var num =100 也不会被销毁
  • 将来 f()的时候，就能够访问到 num 里面的 100
  • 如果将来不需要再获取num了，就把f设置为null

闭包特点

  1. 作用域空间不销毁

    • 优点 因为不销毁 变量也不会销毁 延长变量的生命周期
    • 缺点 因为不销毁 会一直占用内存 多了以后导致 内存溢出

  2. 可以利用闭包在一个函数外部访问其内部的私有变量

    • 优点 可以在函数外部访问内部数据
    • 缺点 必须要一直保持引用，导致函数的执行栈不被销毁

  3. 保护私有变量

    • 优点 可以把一些变量放在函数里面，不会污染全局
    • 缺点 要利用闭包函数才能访问到，比较麻烦

闭包概念

有一个 A 函数，在 A 函数内部返回一个 B 函数
在 A 函数外面有变量引用这个 B 函数
B 函数内部访问着 A 函数内部的私有变量

以上三个条件缺一不可

函数普通调用

  1. call
    语法:跟在函数名后面调用
    函数名.call()
    对象名.函数名.call()
    意义:修改函数内的 this 指向
    参数:
    第一个参数:函数内的 this 指向
    第二个参数:依次传递给该函数的实参
    特点:立即调用函数

  1. apply
    语法:跟在函数名后面调用
    函数名.apply()
    对象名.函数名.apply()
    意义:修改函数内的 this 指向
    参数:
    第一个参数:函数内的 this 指向
    第二个参数:是一个数组或者伪数组都可以，里面的每一项依次传递给函数
    特点:立即调用函数
    特殊作用:改变函数传递参数的方式

  2. bind
    语法:跟在函数名后面调用
    函数名.bind()
    对象名.函数名.bind()
    意义:修改函数内的 this 指向
    参数:
    第一个参数:函数内的 this 指向
    第二个参数:依次传递给函数的实参
    特点:不会立即调用函数，而是返回一个新的函数，就是一个被改变了 this 指向的函数
    特殊作用:改变一些不需要立即执行的函数内的 this 指向

定义变量

  1. let 变量
  2. const 常量

3.1 let/const 和 var 区别

  1. 预解析
    var 定义变量会预解析，可以先访问不报错
    let/const 不会预解析，必须先定义后使用
  1. 变量重名
    var 定义的变量可以重名，只是第二个没有意义
    let/const 不允许在同一作用域下，定义重名的变量
  1. 块级作用域
    var 没有块级作用域
    let/const 有块级作用域
    块级作用域：任何书写{}的地方都会限制变量的使用范围

3.2 let 和 const 的区别

  1. 声明时赋值
    let 可以声明时不赋值
    const 声明时必须赋值
  1. 值的修改
    let 定义的变量可以修改
    const 定义的常量 不能修改
