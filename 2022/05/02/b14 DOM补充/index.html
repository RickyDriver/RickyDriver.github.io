javascript 笔记09-DOM操作补充知识

发表于 2022-05-02 更新于 2023-05-31 分类于 技术
本文字数： 3.6k 阅读时长 ≈ 12 分钟

元素的大小、滚动和位置

clientWidth 内容宽度加上内边距,不包含滚动条
clientHeight 内容高度加上内边距，不包含滚动条

clientLeft:border-left 的宽度
clientTop:border-top 的宽度

offsetWidth:元素的完整宽度
offsetHeight:元素的完整高度

offsetTop:与父元素上边界在 y 轴上的距离
offsetLeft:与父元素左边界在 x 轴上的距离

scrollHeight:可滚动区域的高度
scrollTop:滚去的高度

window 的大小和滚动

大小

outerWidth:整个浏览器的宽度 包含工具栏标签页…
outerHeight:整个浏览器的高度
innerWidth:视窗的宽度
innerHeight:视窗的高度
也可以通过 html 来获取宽高

滚动

window 的滚动位置
scrollX:x 轴滚动的位置
scrollY:Y 轴滚动的位置
注意监听 window 的滚动:window.onscroll=function(){}

也有对应的方法去滚动
scrollBy(x,y):相对于当前位置的(x,y)位置
scrollto(x,y):直接滚动到(x,y)

事件对象

是一个对象数据类型，记录了本次事件的所有信息

事件对象信息-鼠标事件

  1. clientX 和 clientY 相对于视口
  2. pageX 和 pageY 文档
  3. offsetX,offsetY 元素

事件的传播

  • 当一个事件在浏览器中触发的时候
  • 不光在自身元素上触发，是会传播出去的
  • 传播的是什么？传播的是事件的行为

概念:当你在一个元素上触发行为的时候，会按照父级结构的顺序向上传递，一直向上传递到 window

传播的是行为 传播的是行为 传播的是行为

冒泡和捕获

搞清楚概念,当一个事件发生了之后,浏览器先捕获到这个事件,然后事件才冒泡传播

事件冒泡:在事件的传播过程中,从目标到 window 的过程
事件捕获:在事件的传播过程中,从 window 到目标的过程
事件目标:准确触发事件的元素 怎么拿？事件对象.target

怎么在捕获阶段执行？
DOM 0 级不可以
DOM 2 级才可以,addEventListener 的第三个参数实现

  • 默认是 false，表示冒泡阶段触发
  • true，捕获阶段触发

阻止事件传播

  • e.stopPropagation()
  • e.cancelBubble = true
  • try{}catch(err){}

阻止默认事件

  • a 跳转
  • 表单提交

事件委托

核心思想:当事件冒泡的时候,事件也会传递给父级元素,在父级元素设置事件监听,就可以免去给所有的子元素添加监听事件,且能够做一些特殊的事

  • 不用事件委托,使用循环绑定:
    给多个元素绑定相同事件
    但是有缺点 性能，对于添加的元素无效

  • 事件委托来完成
    点击li的时候做一些事
    因为事件会传播，如果我给他的父级ul绑定一个点击事件
    也会因为我点了li而触发

  • 事件对象里面，有一个非常非常非常重要的属性 target 准确记录了触发事件的元素

  • 原则:
    尽可能找距离最近的公共的父级
    尽可能找页面上不动的元素来委托
