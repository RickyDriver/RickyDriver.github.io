元字符–基本字符

  1. 元字符
    所有的文本内容
    特殊符号，用符号来表示一类内容
  2. 标识符
    书写在正则外面，用来修饰正则表达式

基本的元字符

  1. \d表示一位数字
  1. \D表示一位 非数字
  1. \s 表示一位 空白内容
  1. \S 表示一位 非空白内容
  1. \w 表示 一位数字(0-9)字母(a-zA-Z)或者下划线_中的任意一个
  1. \W表示一位 非数字字母下划线
  1. .表示一位不是\n 的字符
  1. \转义字符
  • 把有意义的符号转成没有意义的普通文本
  • 把没有意义的文本转成有意义的符号
    w 没有意义表示字符 w 但是你和\一起写 \w表示一位数字字母下划线

元字符–边界符号

  • ^表示开头
  • $表示结尾

注意:当^和$一起使用的时候，表示的是从开头到结尾

元字符–限定符号

限定内容出现的次数

注意:一个限定符只能修饰限定符前面的一个内容出现的次数
*表示 0 ～多次
前面的内容重复至少 0 次，0 ～+∞ 次

+表示 1 ～无穷次

?表示出现 0 ～ 1 次

{n}表示指定出现 n 次

{n,}表示至少出现 n 次
{0,} 等价于*
{1,} 等价于+

{n,m}出现 n~m 次
{0,1}等价于?

小练习

[]
包含
一个[]可以写多个内容，但是一个[]只占一个字符的位置,表示[]内的任意一个内容都可以
[0-9]等价于\d
[0-9a-zA-Z_]等价于\w

[^]非
一个[^]可以写多个内容，但是一个[^]只占一个字符的位置,表示[^]内的任意一个内容都不可以

- 到 至
需要和[]或者[^]连用

还债

exec

捕获
语法:reg.exec(字符串)
作用:从字符串中 把满足正则条件的部份 取出来
返回值:

  1. 原始字符串中没有满足的
    null
  2. 原始字符串中有满足的
    1. 正则没有()也没有 g
      它的返回值是一个数组
      索引 0 是第一个满足条件的内容
      不管捕获多少次，每次都是从原始字符串的索引 0 开始捕获
    2. 有 g
      它的返回值是一个数组
      索引 0 是第一个满足条件的内容
      第二次捕获是从第一次捕获的结束位置开始向后查询，知道最后没有为止，再下一次又从头开始
    3. 有()
      它的返回值是一个数组
      索引 0 是第一个满足条件的内容
      从索引 1 开始，依次是每个小括号的单独内容
      ()有两个意义:整体，单独捕获 如果不想使用单独捕获，只想作为整体来用需要写成(?:)

两种方式创建的区别

  1. 语法不一样
  2. 书写标识符的区别
  1. 拼接字符串
    字面量不接受拼接字符串
    构造函数可以
  1. 基本元字符的书写

正则的两个特性

  1. 懒惰性
    每一次捕获都是从 0 开始
    解决：全局标识符 g
  2. 贪婪性
    贪婪匹配:能拿多少拿多少 尽可能多的匹配内容
    非贪婪匹配:能拿多少拿多少 尽可能少的匹配内容
    使用非贪婪的限定符号
    在原先的限定符后面再写一个?
    *? 0 ～多次 但是 0 次能解决的问题，就不再多
    +?1 ～多次
    ?0~1 次
    {n,}?
    {n,m}?
