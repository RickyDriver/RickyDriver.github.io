函数的 this 指向

this 指向的疑惑

同一个函数，不同的调用，产生了三种不同的结果

  1. 函数在调用时，js 会默认给 this 绑定一个值
  2. this 的绑定和函数定义的位置无关
  3. this 的绑定和调用方式以及调用位置有关系
  4. this 是在运行时被绑定的

4 个绑定规则
默认绑定
隐式绑定
显式绑定
new 绑定

this 绑定规则一：默认绑定

独立函数调用

this 绑定规则二: 隐式绑定

通过某个对象进行调用

this 绑定规则三: new 绑定

js 中的函数可以当作类的构造函数来使用，也就是可以使用 new 关键字
new 做了那四件事？

  1. 创建一个新的对象
  2. 新的对象会执行 prototype 的连接
  3. 这个新对象会被绑定到函数的 this 上(this 的绑定在这一步完成)
  4. 如果函数没有返回其他复杂数据类型，那就返回这个新对象

this 绑定规则四: 显示绑定

隐式绑定的前提条件，调用的对象内部必须有一个对函数的引用，如果没有这样的引用，在进行调用时，会报找不到的错误

正是对象内的函数引用，间接的让 this 绑定到了这个对象上

我们不希望在 对象内部 存着函数的引用,同时又希望函数的this指向这个对象
js 的所有函数都可以使用 call/bind/apply 方法

call、apply

通过 call 或者 apply 绑定 this 对象，显示绑定后，this 就会明确的指向绑定的对象

bind

如果我们希望一个函数总是显示的绑定到一个对象上（不希望对象里面有该函数的引用，也不希望立即调用该函数），怎么做？

bind 方法，创建一个新的绑定函数(bound function,bf)
绑定函数是一个 怪异函数对象

规则的优先级

  1. 默认规则的优先级最低
  2. 显示绑定优先级高于隐式绑定
  3. new 绑定高于隐式绑定
  4. new 绑定高于 bind
  5. bind 高于 call apply
