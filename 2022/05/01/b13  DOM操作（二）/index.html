DOM 操作(二)

1. 元素的特性 attribute

浏览器在解析 html 元素时，会将对应的 attribute 也创建出来放在相应的对象上
比如 id、class 是全局的 attribute，会有对应的 id、class

1.1 attribute 的分类

  • 标准的 attribute 比如 id class title href value
  • 非标准的 attribute 开发者自定义的 比如 age kfc abc

age=”18”

age = attribute

“18”=property

1.2 attribute 的操作

所有的 attribute 都可以支持如下方法:
ele.hasAttribute(name)- 检查某个特性是否存在
ele.getAttribute(name)- 获取这个特性值
ele.setAttribute(name,value) - 设置某个特性值
ele.removeAttribute(name) - 移除某个特性 值
ele.attributes attr 的对象集合，具有 name、value 属性

attribute 的特征
大小写不敏感(id 和 ID 相同)
他们的值总是字符串类型

2. 元素的属性 property

2.1 元素 property 的操作

  • 对于标准的 attribute,会在 DOM 对象上创建其对应的 property 属性

  • 大多数情况下，他们都是相互作用的

    • 改变 property，通过 attribute 获取的值，会随着改变
    • 通过 setAttribute 设置值，property 的值会随着改变
      但是 input 的 value 修改只能走 attribute

  • 除非特别情况，大多数情况下推荐使用 property 方式，因为他默认情况下是有类型的.

2.2 HTML5 的data-* 自定义属性

h5 的自定义属性

  • 在元素节点上有一个 dataset 的成员
  • 是一个类似 对象 的数据类型
  • 里面存储着所有 data-开头的自定义属性

3.元素的 class、style

动态修改样式的两个选择

  1. 在 css 中写好对应的样式，然后动态操作元素的类名
  2. 动态的修改 style 属性

开发？更推荐用 class，不得不用 style(精准修改某一个的 css 属性的值)

3.1 初体验

3.2 Element-class 用法

  • 元素的 class attribute 对应的 property 并不叫 class 而是 className；
    因为 js 早期(现代的所有高级编程语言里面)不允许使用 class 关键字来作为对象的属性，所以 DOM 的规范就使用了 className
  • 我们可以直接对 className 赋值，但是他会替换整个类名
  • 如果需要添加/移除 单个 class 推荐使用 classList
  • ele.classList 是一个特殊的对象
    ele.classList.add(class) 添加类名
    ele.classList.remove(class) 移除类名
    ele.classList.contains(class) 检查是否有类名
    ele.classList.toggle(class) 切换类名

3.3 Element-style 用法

  • 如果要单独修改某一个 css 属性，可以通过 style 来操作，对于多词(multi-word)属性，使用 camelCase(小驼峰)
  • 如果将值设为空字符串,那么会使用默认样式
  • 多个样式的写法要用到 cssText 但是不推荐
  • 如果要读取非行内样式要通过getComputedStyle(ele).width
  • 兼容ie低版本 ele.currentStyle
