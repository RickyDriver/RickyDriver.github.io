前端开发学习笔记 03 表单、表格

表单

表单的基本结构

一个可以和后台进行数据交互的表单,至少要包含form 标签，输入框，提交按钮

form 元素特有的属性

action 属性

action 用于设置将数据发送给谁，表单的数据提交目标

method 属性

  • get 明文发送(数据通过地址栏传递)
  • post 暗文发送数据(数据不通过地址栏发送)

input 元素

input 元素的表现形式是由该元素的 type 值来决定的

  • type:确定 input 展示为那种类型的输入框
  • name:规定 input 元素的名称，提交表单的时候 name 作为数据一起提交
  • value:input 的值，如果填写的话就作为 input 的默认值
  • readonly:只读，用户不能修改
  • required:设置为必填项，提交表单时，该内容绝对不能为空
  • placeholder:当内容为空时，展示的提示信息

文本域

文本域可以通过css定义宽高，注意不要漏了form标签

表格

一般实际开发除特殊场合外，不推荐用表格来搭建页面结构。

  • table 一个表格
  • caption 表格标题 加粗居中
  • thead 表格的头部区域
  • tbody 表格的主体区域
  • tfoot 表格的底部区域
  • tr 表格的行
  • th 表格的表头单元格 加粗
  • td 表格的单元格 一般是 tbody/tfoot 中 tr 的子元素

表格的嵌套规则

table元素下 只能直接包含 caption,thead,tbody,tfoot

实际尝试后，可以直接用td、tr搭建页面结构，但是这种方法不规范

thead,tbody,tfoot下可以直接包含tr

tr下可以包含的是td

表格的单元格合并!important

单元格的合并不是吞并而是扩大,每次合并单元格后，记得删除多余的那个

rowspan 行合并 下一个 tr 中同样位置的第 n 个
colspan 列合并 同个 tr 中根据 n 来决定

表格相关的一些问题

  1. caption 是否必须
    标题标签不是必须的，对于一个 table 来说，可能存在标题，也可能不存在。
  2. tbody，thead，tfoot 是否必须
    他们都是非必须书写的，对于 tbody 也可以不写，现在的主流浏览器都会自动添加 tbody，但是为了规范，最好加上
  3. tbody，thead，tfoot 元素的顺序是否能改变
    完全可以，只是不利于维护阅读。
