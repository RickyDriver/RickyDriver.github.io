前端开发学习笔记 19 移动端适配

发表于 2021-04-20 更新于 2023-08-13 分类于 笔记
本文字数： 820 阅读时长 ≈ 3 分钟

什么是移动端适配？

  • 移动端开发目前主要包括三类
    • 原生 App 开发(ios android RN uniapp flutter)
    • 小程序(原生小程序 uniapp taro)
    • web 页面
  • 因为目前移动端的设备非常多，所以需要对其进行一些适配
  • 两个概念
    • 自适应:根据不同设备的屏幕大小来自动调整尺寸、大小
    • 响应式:会随着屏幕的实时变动而自动调整，是一种自适应

认识视口

视口的概念：

  • 在一个浏览器中，我们可以看到的区域就是视口
  • fixed 是相对于视口来进行定位的
  • 在 pc 端中，我们不需要对视口进行区分，因为我们的布局视口和视觉视口是同一个

在移动端视口的情况不太一样，

  • 因为手机的屏幕往往比较小，我们可能会希望一个大的网页在移动端可以完整的显示
  • 在默认情况下，移动端的布局视口是大于视觉视口

在移动端，视口划分为三种情况

  • 布局视口(layout viewport)
    • 默认情况下，一个 pc 的网页在移动端是如何展示？
    • 第一：按照宽度为 980px 来布局
    • 第二：为了显示完整的网页，它会对整个页面进行缩小
  • 视觉视口(visual viewport)
    • 默认情况下，按照 980px 来显示内容，右侧会有区域看不到，所以手机端的浏览器会缩放到能够完整显示
    • 那显示在可见区域的这个视口就是视觉视口
  • 理想视口(ideal viewport)
    • 默认情况下的 layout viewport 980px 并不适合我们布局
    • 我们可以对 viewport 进行设置，以满足在正常的移动端窗口布局
    • <meta name="viewport" content="width=device-width, initial-scale=1.0,user-scalable=no,minimum-scale=1.0,maximum-scale=1.0" />

rem+动态的 html 的 font-size

动态 font-size

  1. 媒体查询
    1. 需要针对不同的屏幕编写大量的媒体查询
    2. 如果动态改变尺寸，不会实时更新
  2. 手写 js 设置
    1. 根据 html 的宽度计算出 font-size 大小然后设置到 html 上
    2. 监听页面宽度变化，并重新设置 font-size

适配方案 vw

vw 相比 rem 有很多优势

  1. 不需要去计算 html 的 font-size 大小，也不需要去设置这样一个字体大小
  2. 不会因为设置了 html 的 font-size 而还需要单独设置 body 的 font-size
  3. 不依赖于 font-size 属性，所以不用担心 html 的 font-size 被改掉而导致页面混乱
  4. vw 相比 rem 更加语义化，1vw 刚好是 1/100 viewport 大小
  5. 可以同时具有之前 rem 的所有优点
